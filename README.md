# HolbertonSchool BnB Project

## Description

This project consists of building a simplified clone of the Airbnb platform, named **Hbnb**, following a progressive and modular approach.

The project is divided into multiple parts, each focusing on a specific aspect of software engineering: UML modeling, business logic, API development, authentication, database integration, and frontend implementation.

The main objective is to design and build a complete web application from conception to implementation, while applying industry-standard methodologies such as UML and layered architecture.

---

## Project Objectives

The goals of this project are to:

- Understand and apply UML modeling to describe a software system
- Identify and model a business domain
- Design a clean and scalable application architecture
- Implement backend logic using a layered approach
- Manage authentication and data persistence
- Develop a functional frontend interface
- Work both in team and individually, depending on the project phase

## Architecture Overview

The application follows a layered architecture, separating responsibilities into distinct layers:

- Presentation layer (API / frontend)
- Business logic layer
- Domain layer (business entities)
- Persistence layer (data storage)

This structure ensures maintainability, scalability, and clear separation of concerns.

---

## Methodology

- UML is used as a common language to model the system
- The project follows an incremental and iterative approach
- Code quality and architecture are prioritized over feature quantity
- Manual reviews are conducted at the end of each project part

---

## Technical Constraints

- Project developed following Holberton School guidelines
- Git-based workflow
- Manual code reviews
- Respect of software engineering best practices

---

## Limitations

This project is a simplified and educational version of Airbnb.

Some real-world features are intentionally excluded, such as:
- Online payments
- Messaging system
- Advanced search and filtering
- Refund and cancellation policies
